# ll-humanizer-zh

一个专门给中文文章、短视频脚本、公众号稿、小红书笔记去 AI 味的 Codex Skill。

它的目标不是把文章改得更华丽，也不是重新写一套观点，而是把 AI 写作里常见的老师味、说教感、替读者想、过度总结、报告腔去掉，让内容更像真人写出来的。

## 这个 Skill 是干嘛的

一句话：帮你把 AI 写的东西，改得更像人写的。

它主要处理这些问题：

- 替读者想，比如很多人觉得、你可能以为
- 纠正型表达，比如不是 X，而是 Y、真正重要的是
- 自问自答，比如那能不能？能
- 把读者放低，比如普通人一定要、你只需要
- 冷静分析太多，像报告，不像真实表达
- 泛泛评价，比如效果很好，但不说哪里好

它会尽量保留你的核心观点、信息量和文章结构，只把表达改得更自然、更口语、更平等。

## 新手怎么用

安装好以后，不用先配置，也不用先写个人风格。

你可以直接把文章发给 Codex，然后说：

```text
用 ll-humanizer-zh 帮我把这篇文章去掉 AI 味，保留原意，改得更像真人说话。
```

也可以更简单：

```text
帮我用 ll-humanizer-zh 优化这篇文章，去掉 AI 味。
```

然后粘贴文章就行。

第一次使用时，它默认会用通用模式处理。它不会一上来问你一堆问题，也不会强制你先填写风格模板。

## 它会自动问我问题吗

一般不会。

如果你只是给它一篇文章，让它去 AI 味，它会直接开始处理。

只有在这些情况下，它才需要问你：

- 你明确说想让文章更像你的个人风格
- 你想保存自己的写作风格
- 文章缺少必要上下文，直接改可能会误伤核心观点

大多数情况下，你直接给文章就可以。

## 安装方式

如果你在 Codex 里使用，可以把这个 GitHub 仓库链接发给 Codex，然后说：

```text
帮我安装这个 Skill。
```

Codex 会把它放到你的本地 skills 目录。

常见位置：

```text
~/.codex/skills/ll-humanizer-zh
~/.claude/skills/ll-humanizer-zh
```

安装完之后，重启或刷新 Codex 环境。

之后在对话里输入斜杠，通常就能看到这个 Skill。

## 最佳使用方式

通用版装好就能用。

它检查的是中文 AI 写作里很常见的问题，比如爹味、教条味、替读者想、自问自答、报告腔等等。

但如果你经常写内容，我更建议你多做一步：给它一份你自己的语言风格文档。

通用版解决不像 AI 的问题。

风格文档解决像不像你的问题。

### 第一步：找几篇你自己写的内容

不用多，3 到 5 篇就够。

可以是公众号文章、小红书笔记、朋友圈长文、短视频口播稿，什么形式都行。

关键是两点：

- 这是你自己写的
- 你觉得这就是你的风格

### 第二步：让 Codex 提炼你的语言风格

把这几篇内容发给 Codex，然后说：

```text
这几篇是我自己写的内容，帮我提炼我的语言风格特征。
包括常用句式、用词习惯、段落节奏、标点符号使用方式、情绪表达方式。
请输出成一份风格文档。
```

它会帮你整理出一份语言风格画像。

### 第三步：把风格文档和 Skill 一起用

下次去 AI 味时，把文章和风格文档一起给 Codex，然后说：

```text
请用 ll-humanizer-zh 处理这篇文章。
同时参考我的语言风格文档。
保留核心观点，去掉 AI 味，改得更像我自己写的。
```

这样改出来的文章，不只是没有 AI 味，也会更贴近你的表达习惯。

### 第四步：让风格文档继续进化

第一次用的时候，结果不一定完全合你心意。

你可以手动再改一版。比如改几个词、换几个句式、调整段落顺序、删掉不喜欢的表达。

改完以后，把 AI 修改版和你的最终版一起发给 Codex，然后说：

```text
这是你帮我改的版本，这是我手动修改后的最终版。
请对比两个版本，找出我做了哪些你没想到的调整。
把这些偏好更新到我的风格文档里。
```

这样用几次以后，它会越来越懂你的表达习惯。

## 常用提示词

只去 AI 味：

```text
用 ll-humanizer-zh 帮我优化这篇文章，去掉 AI 味，保留原意。
```

去 AI 味并更口语：

```text
用 ll-humanizer-zh 处理这篇文章。
要求更像真人说话，更口语，少一点说教感，不要改核心观点。
```

检查问题，不直接改全文：

```text
用 ll-humanizer-zh 检查这篇文章哪里有 AI 味。
先只诊断，不要直接改全文。
```

结合个人风格：

```text
用 ll-humanizer-zh 优化这篇文章。
同时参考我的语言风格文档，让它更像我自己写的。
```
